# 제작방식 정본 - 모두일보 콘텐츠 생산

최종 갱신: 2026-06-29 KST

## 핵심 정의

모두일보는 사이트 개발 프로젝트가 아니라 콘텐츠 생산 프로젝트다.

생산 대상은 다음과 같다.

- 기사 원고
- 기사 썸네일/삽화/카드뉴스 이미지
- 유튜브/쇼츠 영상 기획안, 대본, 자막 문안
- 카테고리별 톤앤매너와 편집 기준

## 표준 작업 흐름

1. 주제 접수
2. 카테고리 분류
3. 리스크 등급 판단
4. 출처 확인
5. 기사 구조 설계
6. 초안 작성
7. 사실확인 및 표현 검수
8. 이미지/영상 필요 여부 판단
9. 이미지 프롬프트 또는 영상 스크립트 작성
10. 사용자 승인 요청
11. 출고대기 폴더에 정리

## 실시간 키워드 기반 제작 흐름

판다랭크, 네이버, 구글 등에서 확인한 인기 검색어는 기사 아이디어의 출발점으로 사용한다.

1. 인기 키워드 확인
2. 기사화 가능 여부 판단
3. 정치/법률/의료/금융/범죄/사고/평판 리스크 확인
4. 원출처와 2차 출처 분리
5. 후킹 제목 후보 작성
6. 기사 구조 작성
7. 사실확인 후 출고대기 정리

벤치마킹은 주제, 제목 구조, 독자 반응 포인트까지만 참고한다. 기존 기사 문장, 구성, 독점 취재 내용은 베끼지 않는다.

## 헤드라인 기준

- 독자가 궁금해할 질문형 제목은 사용할 수 있다.
- 사실보다 앞서가는 단정형 제목은 피한다.
- 특정 집단을 조롱하거나 혐오를 유도하는 표현은 피한다.
- 정치적 사안은 중립형 제목과 쟁점형 제목을 함께 검토한다.
- 내부에서는 "어그로"라고 부를 수 있으나, 운영 문서와 외부 명칭은 "트렌드/헤드라인 편집"으로 둔다.

## 리스크 등급

### 낮음

- 행사 안내
- 제품/서비스 소개
- 생활 팁
- 일반 정보성 글

처리: 내부 자료가 충분하면 작성 가능. 그래도 출처와 기준일을 남긴다.

### 중간

- 기업 비교
- 시장 동향
- 정책 변화
- 가격, 혜택, 성능 비교

처리: 최신성 확인 필요. 최소 1개 이상의 신뢰 가능한 출처를 표시한다.

### 높음

- 정치, 법률, 의료, 금융, 범죄, 사고
- 개인/기업 평판
- 분쟁, 의혹, 폭로성 내용

처리: 사용자 승인 전 단정 표현 금지. 필요하면 출처 2개 이상 확인 후 작성한다.

## 카테고리별 톤앤매너

카테고리별 담당자는 같은 기사 구조를 쓰더라도 문체와 우선순위를 다르게 가져간다. 총괄은 최종 검수와 출고 통제를 맡고, 실제 초안은 카테고리 담당자 톤으로 작성한다.

### 트렌드/헤드라인

- 검색어에서 독자가 궁금해할 쟁점 문장을 뽑는다.
- 제목 후보는 질문형, 비교형, 반전형을 섞어 만든다.
- 클릭 유도는 허용하지만 허위, 과장, 조롱, 혐오는 금지한다.

### 사회/생활

- 짧고 명확하게 쓴다.
- 독자가 바로 이해할 수 있는 절차와 영향을 먼저 말한다.
- 불안 조장 표현을 피한다.

### 경제/비즈니스

- 숫자, 기준일, 비교 조건을 분명히 쓴다.
- 기업 홍보문처럼 보이지 않게 사실과 해석을 분리한다.
- "가능성", "전망", "확인 필요"를 구분한다.

### IT/AI/테크

- 기술 용어를 쉬운 말로 풀어쓴다.
- 작동 방식, 기대효과, 한계를 같이 적는다.
- 과장된 혁신 표현을 남발하지 않는다.

### 문화/연예

- 가볍게 쓰되 루머성 표현을 피한다.
- 팬덤, 인물 평판, 사생활 관련 내용은 고위험으로 본다.

### 지역/행사

- 언제, 어디서, 누가, 무엇을, 왜, 어떻게를 우선한다.
- 방문자에게 필요한 정보와 이동/참여 조건을 먼저 둔다.

## 기사 기본 구조

```text
제목:
부제:
카테고리:
기준일:

핵심 요약 3줄:
1.
2.
3.

본문:
1. 무슨 일이 있었나
2. 왜 중요한가
3. 독자에게 어떤 영향이 있나
4. 앞으로 확인할 점

출처/확인 근거:
- 

이미지 필요:
영상 필요:
발행 전 승인 필요:
```

## 이미지 제작 기준

- 기사 내용을 보조하는 이미지로 만든다.
- 실제 인물, 기업 로고, 장소를 임의로 합성하지 않는다.
- 특정 사건 보도에서 실제 사진처럼 오해될 수 있는 AI 이미지는 피하거나 "AI 생성 이미지" 표기를 남긴다.
- 썸네일 문구는 기사 제목을 그대로 반복하지 말고 핵심 질문 또는 핵심 숫자를 강조한다.
- 기사 대표 이미지는 1200 x 675 px, 16:9, 300KB 이하를 권장한다.
- 카드와 홈 화면에서 중앙 기준으로 크롭되므로 핵심 피사체는 가운데에 둔다.
- 정치·사건·범죄·사고 등 고위험 기사에서는 실사풍보다 현대미술형 에디토리얼 일러스트를 우선한다.

## 영상 제작 기준

- 영상은 기사 요약형, 설명형, 쇼츠형으로 나눈다.
- 쇼츠는 첫 3초에 질문 또는 핵심 상황을 둔다.
- 유튜브 긴 영상은 배경 설명, 근거, 결론 순서로 구성한다.
- 자막만 봐도 이해되도록 쓴다.

## 발행 전 체크리스트

- 제목이 사실보다 과장되지 않았는가
- 출처와 기준일이 있는가
- 개인/기업을 부당하게 비난하지 않는가
- 저작권 있는 원문을 베끼지 않았는가
- 이미지가 실제 자료처럼 오해되지 않는가
- 사이트 발행/업로드는 사용자 승인 후 진행되는가
