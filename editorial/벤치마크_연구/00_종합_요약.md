## 1. 네 연구를 관통하는 결론 5가지

1. **모두일보 기존 구조가 이미 정석에 가깝다.** 컴포넌트(Header·BreakingTicker·HeroLead·SectionBlock·RankingList·OpinionStrip·MediaGrid·NewsletterCTA + ArticleCard 5변형)와 본문 가독성(17px·행간 1.9·좌본문+사이드바·다크모드)이 NYT·Guardian·연합·한국일보 벤치마크와 1:1로 대응한다. → 갈아엎을 게 아니라 **보강**이 정답.
2. **정적 export + Cloudflare는 뉴스 SEO에 유리하다.** 본문·메타가 JS 로드 전부터 HTML에 있어 구글이 기사를 식별하기 쉽고 LCP·CLS가 강하다. 약점은 광고 스크립트發 INP 하나뿐. 정적 구조를 유지하며 키운다.
3. **차별화 축 = 중립 + 실용 + 영상 자동화.** 신뢰(투명성 장치)로 중립을 "증명"하고, 사내 9:16 영상 양산 파이프라인으로 "기사→카드뉴스→쇼츠"를 자동화하는 게 신생 매체의 무기다.
4. **네이버는 단기 목표가 아니다.** 입점 문턱 강화로 신생 매체엔 비현실적 → 초기 6개월은 구글·카카오·자체 뉴스레터로 트래픽·실적을 쌓고 그다음 검색제휴 신청.
5. **중립의 실패는 본문이 아니라 "제목 프레이밍 + 따옴표 저널리즘"에서 난다.** 본문이 균형을 지켜도 제목 한 줄·검증 안 된 따옴표가 한쪽 프레임을 확정시킨다. → 출고 전 별도 점검 항목 필수.

## 3. 사이트 코드 반영 백로그 (우선순위)

> 이 단계는 **연구·정리까지** 완료. 아래는 다음 작업 제안이며, 착수는 지시 후. 배포는 항상 승인제.

**P0 — 신뢰·SEO 골격 (검색 노출 직결)**
- [ ] 뉴스 사이트맵 추가(최근 2일·1,000개 이하, 일반 sitemap과 별도)
- [ ] `NewsArticle` + `BreadcrumbList` 구조화데이터를 기사 페이지에 주입
- [ ] 작성자 프로필 페이지 + `ProfilePage` 구조화데이터 + `rel="author"`
- [ ] 기사별 OG 이미지 자동 생성(1200×630, 카카오 중앙 80% 안전영역), 현재 `og.png` 1개뿐
- [ ] 제목 규칙 코드화: `<h1>`=`<title>` 일치, 제목에 날짜 금지, 키워드 front-loading

**P1 — 위계·가독 보강**
- [ ] 콘텐츠 타입 배지(속보/단독/LIVE/오피니언 주황/프리미엄) — 항상 dark: 페어링
- [ ] "오늘의 헤드라인 5선" 텍스트 블록(한국형 고밀도)
- [ ] 본문 내 관련기사 인박스 + 기사 하단 추천

**P2 — 수익·리텐션**
- [ ] 광고 슬롯 placeholder(리더보드/인피드/사이드) — 높이 예약으로 CLS·INP 방어
- [ ] 뉴스레터 구독 동선 강화(`newsletter/` 이미 존재)
- [ ] 모바일 공유 sticky 하단바

**P3 — 정적 export 친화**
- [ ] 페이지네이션(무한스크롤 대신 정적 호환)

**상시 — 보안**
- [ ] next@15.1.6 보안 취약점(CVE 경고) 패치 업그레이드 검토 (별도 보고 후 진행)

## 4. 편집국(콘텐츠) 측 즉시 적용

- 기자용 **표준 기사 템플릿 + 출고 전 체크리스트**(01 문서) → 편집 인격 30종 운영에 바로 장착
- **중립 규약 10개조 + 표현 대조표**(03 문서) → 정치/사건/선거 기사 출고 전 강제 점검

## 5. 저작권·윤리 준수 메모

모든 벤치마크 문서는 **기존 기사 본문 복제 없이** 작성됐다. 예문은 전부 새로 만든 가상 예문이고, 출처는 매체명/스타일북명으로만 귀속 표기했다(AP Stylebook, Reuters Trust Principles, BBC Editorial Guidelines, NPR Ethics, SPJ, Trust Project, 한국 신문윤리강령 등). 모두일보 [00_START_HERE](../00_START_HERE.md)의 "기존 언론사 기사 본문을 베껴 쓰지 않는다" 원칙과 일치.
