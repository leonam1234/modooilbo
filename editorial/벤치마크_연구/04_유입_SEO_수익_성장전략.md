## 1. 뉴스 SEO

검색 유입은 "기사 하나하나가 색인되기 쉬운 구조인가"에서 갈린다. 메이저 매체가 공통으로 지키는 규칙.

### 1-1. 제목(headline) — 키워드 × 클릭 × 정직성

- **길이**: 110자 초과 금지(Google 권장). 한국어는 32~45자 선에서 핵심 키워드를 앞쪽에 배치.
- **5W 우선**: 누가·무엇을·언제·어디서·왜를 제목 안에서 답하면 검색·클릭 모두 유리.
- **HTML 일치**: 화면의 `<h1>`과 HTML `<title>`을 일치(또는 거의 일치)시킨다. Google 크롤러는 헤드라인 근처 본문으로 기사 본문을 식별하므로 `<h1>`은 본문 위에 명확히.
- **날짜·숫자 남용 금지**: 제목에 날짜를 넣지 말 것(Google 권장). 앵커 텍스트에 선행 숫자 지양.
- **낚시(clickbait) 금지**: 과장·왜곡 제목은 Google News 정책 위반이자 신뢰도 하락. "정직한 호기심"이 기준.
- **모두일보 룰**: 같은 기사라도 (a) 화면 노출 제목과 (b) `<title>`/OG 제목을 분리 운영 가능. 화면은 편집 감성, `<title>`은 검색 키워드 충실형.

### 1-2. 메타 설명(description)

- 검색 스니펫·SNS 카드의 기본 재료. 기사 첫 문단을 그대로 쓰지 말고 **80~120자 요약**을 별도 작성.
- 핵심 키워드 1~2개 자연스럽게 포함, 본문과 모순 없게.

### 1-3. URL 구조

- **안정적·영구적**: `/article/[slug]/` 형태. 한 번 발행한 URL은 바꾸지 않는다(변경 시 301 영구 리다이렉트).
- `&ID=` 같은 파라미터 URL 금지, 리다이렉트 최소화, meta refresh 금지.
- 모바일·데스크톱 리다이렉트 동일하게.
- 정적 export 환경이므로 `trailingSlash: true`(현재 설정) → `/article/123/index.html` 폴더 구조로 안정적.

### 1-4. 발행·수정 시각 표기

- **화면 표기**: 헤드라인과 본문 사이에 "발행 시각" 하나를 명확히 노출. 수정 시 "수정 시각" 별도 표기.
- **구조화데이터**: `datePublished` / `dateModified`를 ISO 8601(예 `2026-06-29T14:30:00+09:00`)로. 한국 매체는 KST(+09:00) 오프셋을 반드시 포함.
- **어뷰징 금지**: 실질 내용 변경 없이 시각만 갱신해 상단 노출 노리는 행위는 Google News 정책 위반(노출 패널티).

### 1-5. 구조화데이터(schema.org)

기사 페이지마다 JSON-LD 삽입. 정적 export라면 빌드 시점에 HTML에 박아넣어야 한다(클라이언트 JS 로드 전후 모두 본문·메타가 HTML에 존재해야 함 — Google이 강조).

**NewsArticle 필수·권장 필드**
- `headline`(110자 이내), `image`(가능하면 3종 — 1:1, 4:3, 16:9), `datePublished`, `dateModified`
- `author`(이름 + **작성자 프로필 URL** — E-E-A-T 핵심), `publisher`(이름 + 로고)
- `mainEntityOfPage`, `description`

**BreadcrumbList**: 홈 > 카테고리 > 기사 경로를 schema로. 검색결과 빵부스러기 노출.

**ProfilePage(작성자 페이지)**: 2024년 8월 Google이 권장 구조화데이터에 추가. 작성자 페이지에 적용.

> 모두일보 구현: `src/app/article/` 레이아웃에 JSON-LD `<script type="application/ld+json">`를 서버 컴포넌트에서 렌더해 정적 HTML에 포함. 기사 데이터(제목/저자/시각/이미지)를 단일 소스에서 받아 schema·OG·화면을 한꺼번에 채운다.

### 1-6. 뉴스 사이트맵(news sitemap)

일반 사이트맵과 **별도**로 운영. Google News 색인 속도에 직결.

- 한 파일에 `<news:news>` 최대 1,000개. 초과 시 분할.
- **최근 2일 이내** 발행 기사만 포함. 2일 지나면 해당 항목 제거(또는 news 메타 제거).
- 필수 태그: `<news:publication>`(name + language), `<news:publication_date>`(W3C 포맷), `<news:title>`(저자·매체·날짜 없는 순수 제목).
- 새 기사 나올 때마다 **같은 파일 갱신**(매번 새 파일 만들지 말 것).
- `<news:name>`은 Google News에 등록될 매체명과 정확히 일치.

> 모두일보 구현: 현재 `sitemap.ts`(일반)는 유지하고, `public/news-sitemap.xml`을 빌드 스크립트로 **최근 2일 기사만** 생성하는 별도 라우트/스크립트 추가. `robots.ts`에 두 사이트맵 모두 명시.

### 1-7. Core Web Vitals (정적 export의 강점 살리기)

| 지표 | 의미 | 목표 |
|---|---|---|
| LCP | 최대 콘텐츠 표시 | < 2.5초 |
| INP | 상호작용 응답성 | < 200ms |
| CLS | 시각적 안정성 | < 0.1 |

- 2026년 Google은 페이지별이 아닌 **사이트 전체(holistic)** 평가로 전환. 한 곳이 나쁘면 전체가 끌려간다.
- INP가 가장 흔한 탈락 지점(웹 전체 43% 실패). 광고 스크립트·무거운 JS가 주범.
- **정적 export + Cloudflare CDN은 LCP·CLS에 절대 유리.** 살릴 포인트:
  - 기사 대표 이미지에 명시적 `width/height`(CLS 방지), `loading="lazy"`는 본문 하단 이미지에만.
  - 광고 슬롯은 **고정 높이 컨테이너**로 예약(광고 로드 전후 점프 방지).
  - 폰트는 `font-display: swap` + 서브셋. 서드파티 스크립트는 `defer`/지연 로드.

## 2. Google News / Discover와 E-E-A-T

### 2-1. 노출 요건(2025~2026 현재)

- **신청 절차 폐지**: Google은 2025년 News 신청 프로세스를 종료. 품질 기준을 충족하면 **자동 발견·색인**된다. → "신청"이 아니라 "색인되기 좋은 구조 + 꾸준한 발행"이 답.
- 전제 조건: 명확한 발행 시각, NewsArticle 구조화데이터, 뉴스 사이트맵, 안정적 URL, 오리지널·사실확인 콘텐츠.

### 2-2. E-E-A-T를 3개 층에서 증명

Google은 경험·전문성·권위·신뢰를 **페이지/작성자/매체** 3층에서 본다. 한두 층만 잘해선 부족.

1. **페이지 층**: 개별 기사 품질, 출처 인용, 정확성. 정정·반론 표기.
2. **작성자 층**: 기자 크리덴셜·경력·이력. → **작성자 프로필 페이지 필수.**
3. **매체 층**: 편집 기준·윤리강령·연락처·소유구조 공개. (모두일보엔 이미 `ethics/`, `about/`, `contact/` 페이지 존재 — 채워 넣을 것.)

### 2-3. 작성자 프로필 페이지(가장 자주 빠뜨리는 항목)

- 각 작성자 전용 URL(예 `/author/[name]/`): 사진, 약력(왜 이 주제에 자격이 있는가), 담당 분야, 작성 기사 목록.
- 기사 본문의 작성자 이름은 프로필로 클릭 이동. 작성자의 SNS(LinkedIn/X)는 `rel="author"` 표기 → Google이 외부 프로필과 교차검증.
- 작성자 페이지에 ProfilePage 구조화데이터.

### 2-4. Discover & 팔로우(2025년 9월 대형 업데이트)

- 2025년 9월 Google은 Discover에 **퍼블리셔 팔로우 + 전용 프로필 페이지**를 도입.
- Knowledge Graph 보유 + 정기 발행 매체는 프로필 페이지 대상. → 모두일보를 Knowledge Graph에 인식시키려면 일관된 매체명·로고·`sameAs`(공식 SNS) Organization 구조화데이터를 홈에 둘 것.
- Discover는 검색어가 아니라 **관심사 피드**. 강한 대표 이미지(1200px 이상 고화질)와 클릭 유도형(낚시 아닌) 제목이 유리.

## 3. 네이버 뉴스 생태계

한국 뉴스 트래픽의 현실: 검색·뉴스 소비가 네이버에 크게 쏠려 있다(국내 포털 점유율 약 60%). Google News 최적화만으로는 한국 독자 도달이 제한된다.

### 3-1. 제휴 3단계

| 단계 | 방식 | 트래픽·수익 |
|---|---|---|
| **검색제휴** | 아웃링크(클릭 시 자사 페이지로 이동) | 가장 기본형. 네이버 검색 결과에 노출, 트래픽은 자사로 들어옴 |
| **뉴스스탠드 제휴** | 아웃링크 | 뉴스스탠드 영역 노출 |
| **콘텐츠제휴(CP)** | 인링크(네이버 안에서 열람) | 네이버가 기사 구매, 전재료·광고비 지급. 도달 최대지만 트래픽은 네이버에 귀속 |

### 3-2. 2025~2026 입점 현실 (중요)

- 네이버는 2025년 6월 **뉴스제휴위원회** 출범, 신규 입점 심사 재개.
- 신규 평가는 **정량평가 50% 이상** 비중.
- **입점 문턱 상승**: 과거 60점(검색제휴)·80점(콘텐츠제휴) → **80점·90점**으로 강화.
- 신생 매체에 검색제휴조차 단기간엔 어렵다는 현실 인식 필요. → **초기엔 Google·SNS·자체 채널 중심으로 트래픽을 쌓고, 정량 실적(발행량·자체 트래픽·체류시간)을 만든 뒤 제휴 신청** 순서.

### 3-3. 네이버 노출 시 유의점

- 네이버는 제목 어뷰징·중복·우라까이(베껴쓰기)에 제재. 검색제휴라도 어뷰징 적발 시 제재·계약해지.
- 네이버 검색에 잡히려면: 명확한 발행/수정 시각, 본문 구조, 자체 도메인 안정성. 콘텐츠제휴 매체만 검색하는 기능도 도입되어 CP 미보유 매체의 노출은 더 제한적.

### 3-4. 모두일보 전략

- 단기: 네이버 의존 X. 자체 도메인 SEO + Google + 카카오/SNS로 직접 트래픽.
- 중기: 발행 6개월~1년 정량 실적 쌓은 뒤 **검색제휴** 신청.
- 데이터랩(검색어 트렌드)·네이버 검색을 **선택·기획 도구**로 활용(아래 5장).

## 4. SNS·메신저 유입 — 오픈그래프

한국은 카카오톡 공유가 강력한 유입원. OG 카드 품질이 클릭률을 좌우한다.

### 4-1. 핵심 태그(기사마다 동적)

```html
<meta property="og:title" content="검색·클릭형 제목" />
<meta property="og:description" content="80~120자 요약" />
<meta property="og:image" content="https://modooilbo.com/og/article-123.jpg" />
<meta property="og:type" content="article" />
<meta property="og:url" content="https://modooilbo.com/article/123/" />
<meta property="og:site_name" content="모두일보" />
<meta property="article:published_time" content="2026-06-29T14:30:00+09:00" />
<meta name="twitter:card" content="summary_large_image" />
```

### 4-2. 이미지 규격

- **범용 권장 1200 × 630px (1.91:1)**, JPEG/PNG, 1MB 이하. Facebook·X·슬랙·디스코드·카카오 등 호환.
- **카카오톡은 2:1로 잘림** → 텍스트·로고는 **중앙 80% 안전영역**에 배치(상하 잘려도 핵심 보존).
- **파일명에 한글 금지**(공유 시 깨짐). 영문/숫자만(`article-123.jpg`).
- 기사별 OG 이미지가 없으면 대표 이미지 폴백, 그것도 없으면 `public/og.png`(현재 보유) 사용.

### 4-3. 캐시·검증

- 변경 후 미반영은 캐시 탓. **Facebook Sharing Debugger**로 재스크랩, **카카오 디벨로퍼스 링크 스크랩**으로 점검.
- 이미지 URL에 버전 쿼리(`?v=2`)를 붙이면 강제 갱신.

> 모두일보 구현: 기사별 OG 이미지를 **빌드 시 자동 생성**(제목+대표이미지 합성)하면 클릭률이 크게 오른다. 정적 export면 빌드 스크립트에서 OG 이미지를 일괄 생성해 `public/og/`에 저장.

## 5. 속보·실시간 키워드 운영

### 5-1. 트렌드 포착 도구

- **Google 트렌드 '실시간 인기'**: 검색 급증 + 뉴스 연관 키워드 표시. (Trends API 알파 공개 — 자동 수집 가능.)
- **네이버 데이터랩**: 검색어 트렌드·쇼핑 인사이트·지역/성별/기기별. 한국 검색 의도 파악의 핵심.
- **판다랭크 등 키워드 도구**: 검색량·경쟁도 보조.

### 5-2. 기사화 판단 기준

급상승 키워드가 떴다고 다 쓰는 게 아니다. 다음을 통과할 때만:
1. **사실 확인 가능한 1차 출처**가 있는가(공식 발표·당사자·문서)?
2. 모두일보가 **추가 가치**(맥락·검증·정리)를 더할 수 있는가?
3. 편집 기준·윤리에 부합하는가?

### 5-3. 어뷰징·우라까이와의 선 긋기 (모두일보 정체성)

- **우라까이(베껴쓰기) 금지**: 타사 기사 재배열·짜깁기로 키워드만 노리는 것은 네이버·Google 모두 제재 대상이자 신뢰 파괴.
- **시각 어뷰징 금지**: 내용 변경 없는 시각 갱신 금지(1-4 참조).
- **중복 양산 금지**: 같은 사안 동일 기사 다중 발행 금지.
- 모두일보 룰: 트렌드는 **주제 선택·각도 설정** 도구일 뿐, 본문은 자체 취재·검증·정리로 채운다. 출처는 항상 매체/기관명으로 명기.

## 6. 독자 묶기 (리텐션)

검색·SNS 유입은 1회성. **소유 채널(owned channel)** 로 묶어야 자산이 된다. 뉴스레터가 가장 효율적인 구독 전환 깔때기.

### 6-1. 뉴스레터 (최우선 무기)

벤치마크(뉴욕타임스):
- 3단 구조 — (1) **무료 데일리 브리핑**(습관·인지 형성, 예 'The Morning') → (2) **주제별 뉴스레터**(관여 심화) → (3) **유료 구독자 전용**(서사형 독점).
- The Morning 구독자의 **42%가 6개월 내 유료 전환**(전략적 페이월 트리거).
- 뉴스레터를 통한 구독 획득 비용이 **SNS 캠페인보다 58% 저렴.** → 소유 채널의 효율.
- 2025년 뉴스레터 기여가 NYT 디지털 매출 성장의 31% 차지.

**모두일보 적용**
- **창간 즉시 무료 데일리/위클리 브리핑** 1종 시작(현재 `newsletter/` 페이지 존재 — 구독 폼 연결).
- 발행 주기 고정(매일 아침 또는 주 2~3회), 구성 = 헤드라인 5~7개 + 한 줄 코멘트 + 핵심 1건 심층.
- 모든 기사 하단·중간에 구독 CTA. 발송 = Cloudflare 환경이면 외부 ESP(Buttondown/Beehiiv/Stibee 등) 연동.

### 6-2. 푸시

- 웹푸시(브라우저)로 속보 알림. 단, 남발 금지(구독 해지 유발). 속보·관심 카테고리 한정.

### 6-3. 멤버십·구독 모델 (사례)

- **NYT(번들형)**: 뉴스 + 게임·요리·오디오·스포츠·리뷰를 묶은 올액세스. 구독자 50%+가 번들/복수상품 → 더 오래·더 많이 결제. 번들은 리텐션의 핵심.
- **Guardian(후원형)**: 페이월 없이 **일회성 후원 → 정기 멤버십** 전환. 후원자를 뉴스레터 깔때기에 넣어 정기 전환. 2025년 레거시 중 구독 성장 가속 1위.
- **동적 페이월**: 2020년 대비 4배 늘어 22% 매체가 사용. 독자 행동(읽은 기사 수·재방문)에 따라 페이월 노출을 조절.

**모두일보 적용(단계적)**
- 창간기: 페이월 X. Guardian형 — **후원 + 무료 뉴스레터로 관계 자산** 축적.
- 성장기: 동적 페이월 or 멤버십(광고 제거·독점 뉴스레터·심층 리포트) 도입 검토.

## 7. 수익 모델

산업 우선순위(2025): **구독·멤버십 76% > 디스플레이 광고 68% > 네이티브 64%.** 트래픽이 줄어도 구독은 성장 중. 단일 수익원 의존은 위험 — **다각화**가 정답.

### 7-1. 광고

- **디스플레이/배너**: 가장 기본. 단 CWV·UX를 해치지 않게 슬롯 높이 예약, 광고 수 제한.
- **네이티브/브랜디드 콘텐츠**: 가장 빠르게 성장. 에디토리얼 형식이라 참여·전환 높음. **반드시 '광고'·'협찬'·'AD' 명확히 분리 표기**(이 표기가 신뢰·법적 안전의 핵심).
- 정적 export 환경: 광고는 클라이언트 스크립트로 삽입되므로 INP 영향 주의(지연 로드).

### 7-2. 구독·멤버십·후원

- 6장 참조. 모두일보 초기 핵심은 **후원 + 뉴스레터**(진입장벽 낮고 신뢰 자산화).

### 7-3. 제휴(affiliate)·라이선스·이벤트·리포트

- 제품 리뷰·가이드형 기사에 제휴 링크(표기 필수).
- 콘텐츠 라이선싱, 유료 리포트(산업 분석), 오프라인/온라인 이벤트.

### 7-4. 영상(유튜브/쇼츠)

- 기사 자산을 쇼츠로 재패키징해 새 유입 + 광고 수익(8장).

### 7-5. 다각화 원칙

성공 매체는 광고+구독+제휴+협찬+라이선스+이벤트를 **조합**한다. 모두일보 1년차 권장 믹스: **후원/뉴스레터(관계) + 디스플레이(현금흐름) + 네이티브(고단가) + 쇼츠(유입)**.

## 8. 영상·쇼츠 패키징

방송사 공식 뉴스 유튜브 분석(2021.11~2024.2, 숏폼 17,338건)에서 보듯 숏폼은 뉴스 생산양식 자체를 바꿨다. 핵심은 **기사 → 카드뉴스 → 쇼츠**로 자산을 재활용하는 파이프라인.

### 8-1. 전환 공식

| 단계 | 산출물 | 포인트 |
|---|---|---|
| 1. 기사 | 본문·대표이미지 | SEO·구독 깔때기 |
| 2. 카드뉴스 | 5~8장 슬라이드 | 핵심 1장 = 1메시지, SNS/인스타 유입 |
| 3. 쇼츠 | 30~60초 세로(9:16) | 검색 밖 신규 도달 |

### 8-2. 쇼츠 제작 룰

- **첫 3초가 전부**: 결론·후킹 멘트를 맨 앞에. 클라이맥스를 도입부에 배치.
- **썸네일·자막 멘트**: 호기심 자극하되 사실 기반(낚시 X).
- 세로 9:16, 최대 3분 가능하나 뉴스는 **30~60초** 권장.
- 한 기사 = 카드뉴스 1세트 + 쇼츠 1편을 기본 산출물로 정형화하면 양산 가능.

> 모두일보엔 사내 영상 자동화 역량(TTS→영상→BGM, 9:16 양산 파이프라인)이 이미 있으므로, **기사 발행 → 쇼츠 자동 생성** 연결이 강력한 차별점.

## 9. 측정 지표

"무엇을 보느냐"가 전략을 정한다. 트래픽 허수보다 **관계·전환 지표** 중심.

### 9-1. 핵심 지표

| 지표 | 무엇을 말하나 | 비고 |
|---|---|---|
| PV / UV | 도달 규모 | 허수 주의 |
| **체류시간(평균/기사별)** | 콘텐츠 실질 가치 | 검색·신뢰 신호와 연동 |
| **스크롤 뎁스** | 본문 끝까지 읽는가 | 광고·CTA 배치 근거 |
| **재방문율** | 충성도 | 리텐션 핵심 |
| **뉴스레터 구독 전환율** | 유입→관계 | 최우선 KPI |
| **뉴스레터 오픈율 / 클릭율** | 관계 건강도 | 오픈 30%+, 클릭 5%+ 목표선 |
| **유입 채널 비중** | Google/네이버/SNS/직접 | 의존 분산 점검 |
| **구독·후원 전환율 / LTV / 이탈률** | 수익 건강 | 성장기 핵심 |
| Core Web Vitals | 기술 품질 | LCP<2.5s, INP<200ms, CLS<0.1 |

### 9-2. 보는 법

- 창간기: **체류시간·재방문·뉴스레터 구독 전환**에 집중(절대 PV 경쟁 X).
- 성장기: **구독/후원 전환·LTV·이탈률**로 이동.
- 도구: GA4(이벤트·스크롤·체류) + Search Console(검색 노출·CTR·색인) + Cloudflare Web Analytics(서버리스, 쿠키리스).

## 10. 90일 성장 로드맵

기술 전제: Next.js 정적 export + Cloudflare Pages. 현재 `robots.ts`·`sitemap.ts`·`icon.png`·`og.png`·`_headers`·`ethics/about/contact/newsletter` 페이지 보유.

### 단계별 실행 표

| 구간 | 목표 | 핵심 실행 항목 |
|---|---|---|
| **창간 ~ 30일**<br>(기반·신뢰) | 색인 가능한 구조 + 신뢰 자산 | • 기사 페이지 **NewsArticle JSON-LD**(저자·시각·이미지) 빌드 시 삽입<br>• **작성자 프로필 페이지**(`/author/[name]/`) + ProfilePage 구조화데이터<br>• 홈에 **Organization 구조화데이터**(매체명·로고·sameAs)<br>• `ethics`·`about`·`contact` 페이지 실제 내용 채우기(편집윤리·소유구조·연락처)<br>• **뉴스 사이트맵**(최근 2일, `<news:*>`) 생성 + `robots.ts`에 등록<br>• 기사별 **OG 메타 동적화** + OG 이미지(1200×630, 영문 파일명)<br>• **무료 데일리/위클리 뉴스레터** 1종 + 구독 폼 가동(ESP 연동)<br>• GA4 + Search Console + Cloudflare Analytics 연결<br>• 발행 리듬 확립(하루 최소 N건 자체 콘텐츠) |

| **61 ~ 90일**<br>(수익·리텐션) | 수익 다각화 + 관계 심화 | • **후원(일회성→정기) + 멤버십** 도입(Guardian형)<br>• 디스플레이 광고 슬롯 도입(높이 예약·INP 영향 최소화)<br>• 동적 페이월/구독 모델 검토(독점 뉴스레터·리포트)<br>• **네이버 검색제휴 신청 준비**(6개월 실적 축적 시작 — 정량 비중 50%+)<br>• 쇼츠 채널 정착 → 유튜브 광고·유입 루프<br>• 지표 리뷰 정례화: 체류·재방문·구독전환·LTV·CWV<br>• Discover 팔로우/프로필 대비(Knowledge Graph 인식용 일관 브랜딩) |

### 기술 체크리스트 (즉시 점검)

- [ ] 기사 HTML에 `<h1>`(본문 위) = `<title>` 일치, 110자 이내
- [ ] 기사마다 NewsArticle JSON-LD가 **정적 HTML에 포함**(JS 로드 전 존재)
- [ ] `datePublished`/`dateModified` ISO 8601 + **KST(+09:00)**
- [ ] 작성자 프로필 페이지 + ProfilePage 구조화데이터 + `rel="author"` SNS
- [ ] 홈 Organization 구조화데이터(logo·sameAs)
- [ ] 뉴스 사이트맵(최근 2일, 1,000개 이하, `<news:name>` = 등록 매체명) + `robots.ts` 등록
- [ ] OG 태그 동적(title/description/image/url/type=article), 이미지 1200×630·영문 파일명·중앙 안전영역
- [ ] CWV: 이미지 width/height 명시, 광고 슬롯 높이 예약, 폰트 swap+서브셋, 서드파티 defer
- [ ] URL 영구 고정(`trailingSlash` 유지), 변경 시 301
- [ ] GA4·Search Console·Cloudflare Analytics 연결
- [ ] 뉴스레터 구독 폼 → ESP 연동 동작 확인
- [ ] 광고/협찬 콘텐츠 '광고' 분리 표기 컴포넌트
