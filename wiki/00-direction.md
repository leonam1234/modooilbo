# 00 · 방향성 (Direction)

> 이 문서는 시그널저널의 **존재 이유와 의사결정 기준**입니다. 디자인·기능·콘텐츠 판단이 애매할 때 여기로 돌아오세요.

## 1. 브랜드 정체성
- **이름**: 시그널저널 (Signal Journal)
- **슬로건**: *"노이즈 속에서 신호를 읽다"*
- **의미**: 정보 과잉(노이즈) 시대에, 속도·클릭 경쟁이 아니라 **의미 있는 신호(signal)** — 검증된 사실과 깊이 있는 분석 — 을 전달한다.
- **워드마크**: `시그널` + `저널`(저널은 시그널 레드) + 정사각 `S` 마크. → [02-design-system](02-design-system.md)

## 2. 미션 / 비전
- **미션**: 신뢰할 수 있는 사실과 맥락으로 더 나은 공론장을 만든다.
- **비전**: 속도가 아니라 **신뢰**로 이기는 디지털 언론. 독자의 시간을 존중하는 저널리즘.

## 3. 포지셔닝 & 차별화
국내 주요지(조선·중앙·한겨레) 벤치마킹에서 도출한 **의도된 차별점**:

| 축 | 일반적 국내 언론 | 시그널저널의 선택 |
|----|------------------|-------------------|
| 액센트 컬러 | 매체별 블루/레드 혼재 | **단일 "시그널 레드"** 하나로 강한 브랜드 각인 |
| 타이포 | 대개 고딕 일변도 | **본문 Pretendard(산세) + 헤드라인 명조(세리프)** 듀얼 |
| 정보 밀도 | 매우 높음(클릭 최대화) | **약간 낮춘 밀도 + 여백** → 모던·프리미엄 |
| 다크모드 | 약하거나 앱 한정 | **정식 시스템 인지 다크모드** (웹 기본 지원) |
| 수익모델 | 하드 페이월 경향 | **독자 후원·구독 우선**, 콘텐츠 개방 지향 |
| 오피니언 | 보조 영역 | **1급 배치** (홈 전용 오피니언 스트립) |

## 4. 편집 원칙 (핵심 가치)
1. **정확성** — 사실 확인 우선, 정정·반론 보도 원칙. ([/ethics](../src/app/ethics/page.tsx))
2. **독립성** — 광고가 아닌 독자의 힘. 자본·권력으로부터 독립.
3. **공공성** — 콘텐츠 개방, 공론장 기여.
4. **혁신** — 가장 현대적인 방식(웹/데이터/멀티미디어)으로 전달.

## 5. 타깃 독자
- 디지털 우선·모바일 중심의 정보 적극 소비층.
- 깊이 있는 분석·오피니언·데이터 저널리즘을 선호하는 독자.

## 6. 콘텐츠 전략
- **8개 섹션**: 정치·경제·사회·국제·문화·스포츠·테크·오피니언. → [03-content-model](03-content-model.md)
- **속보 티커** + **메인 헤드라인(리드)** + **섹션 블록** + **많이 본 뉴스** + **오피니언** + **포토·영상** + **뉴스레터**.
- 멀티미디어(포토/영상)와 오피니언을 1급 콘텐츠로 취급.

## 7. 비즈니스 방향
- **후원·구독** (무료 / 디지털 멤버십 9,900원 / 후원 서포터) → [/subscribe](../src/app/subscribe/page.tsx)
- **뉴스레터**(모닝 시그널 등) 로 재방문·구독 전환.
- **광고·제휴**(디스플레이/네이티브/스폰서십/브랜디드) → [/advertise](../src/app/advertise/page.tsx)
- **인재채용·기자모집** 으로 뉴스룸 확장 → [/careers](../src/app/careers/page.tsx)

## 8. 제품 로드맵
| 단계 | 상태 | 내용 |
|------|------|------|
| **Now — 프로토타입** | ✅ 완료 | 전 페이지 + 더미데이터 62건 + 디자인시스템 + CF 정적 배포 |
| **Next — 실데이터화** | ⬜ | Headless CMS(예: Sanity/Contentful) 또는 자체 API로 기사 연동, Cloudflare Images 로더 |
| **Then — 동적 기능** | ⬜ | 인증(로그인/회원), 댓글, 서버 검색(전문 검색), 개인화 추천 → `@opennextjs/cloudflare`(Workers) 승급 |
| **Scale** | ⬜ | 결제(구독 정산), 뉴스레터 발송, 애널리틱스, 커스텀 도메인 |

→ 승급 경로 상세: [06-deployment](06-deployment.md)

## 9. 의도된 비범위 (현재)
- 데이터·기자·연락처·통계는 **모두 가상(데모)**.
- 폼(채용/문의/제보/로그인 등)은 **클라이언트 데모** — 백엔드 전송 없음.
- 이미지는 **picsum.photos 플레이스홀더**.
- 검색은 **클라이언트 텍스트 매칭**(전문 검색 아님).

> 이 비범위들은 "다음 단계"에서 실제 연동으로 대체되도록 의도적으로 경계를 그어둔 것이다. 데모라고 대충 만든 게 아니라, **실연동 지점이 명확하게 분리**되어 있다.
